Web源码扩展

Web源码在安全测试中是非常重要的信息来源，可以用来代码审计漏洞也可以用来做信息突破口，其中Web源码有很多技术需要简明分析。

图片来自于SamXara

WEB源码目录结构

需要至少了解后台目录，模板目录，数据库目录，数据库配置文件的目录结构（了解这些架构便于理解）

目录

部分目录

admin目录表示是后台路径
data目录是处理数据
install目录是处理安装的
member目录是处理会员的
template目录是模板文件（界面为了美观使用了相关模板）
数据库配置文件记录了数据库和相关网站的关联（一般会在includes之类的目录下confing.php之类的）
数据库配置文件会有数据库账号密码之类的相关信息

WEB源码脚本类型

拿到源码后得知道源码的类型，因为不同的源码类型会在数据库存储，解释或编译型和安全方面产生不同。

脚本语言类型有 ASP PHP ASPX JSP JAVAWEB Python

学习网站推荐：Web安全学习笔记

不同脚本语言可能产生的漏洞有Web安全学习笔记

WEB源码应用分类

WEB源码的应用，即web源码的作用
因为源码的功能会决定漏洞的类型

WEB源码其他说明

框架或无框架

有点源码用框架写的，所以得去了解框架的漏洞

如果不是框架写的，也可以找漏洞（应该说更容易找漏洞了）

CMS识别

识别该网站基于什么软件搭建的，就可以方便识别漏洞或根据该软件别人公开的漏洞进行攻击

开源或内部

开源或内部

开源：网上公开了的源码（针对性攻击）

内部：网上未公开的源码（用基本知识去找漏洞）

总结

关注分类和脚本类型估计出可能存在的漏洞（框架漏洞例外)，获取源码后1.在本地搭建网站进行安全测试和2.在代码中去做安全测试 代码审计或是3.分析其目录的工作原理（通过分析其工作原理得到数据库备份，bak文件等）

  1. 找到一个网站A，通过地址的后缀分析出该网站的源码类型（这里是asp文件）
  2. 根据网站分析出是用啥XYCMS的，去搜索得到源码
  3. 找到源码的数据库路径文件，找到ASP种特有的xycms.mdb（asp特有的文件），该数据库文件会存有账号密码的相关信息
  4. 然后知道了普通的XYCMS源码的路径，数据库文件所在位置的路径，然后在A的地址栏出加上那个数据库文件的路径，就是尝试去碰碰看A的数据库文件是否也在那个路径下（因为是同个源码，可能性较大）
  5. 然后真的那个路径下时，就可以下载到相关的文件了
  6. 然后你就可以得到账号密码的相关信息了
  7. 然后就去后台输密码，就可以了（其中数据库文件中的密码可能加密，此时可以找在线的密码解密的网站解密再输）
