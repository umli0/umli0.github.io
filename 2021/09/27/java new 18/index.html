MyBatis

什么时MyBatis（封装，并扩展了JDBC的功能）
(1) 是apache的一个开源项目iBatis，2010年这个项目有apache迁移到了google，改名为MyBatis，13年迁移到了GitHub

(2) MyBatis是一个优秀的持久层（数据库端）框架，它对jdbc的操作数据库的过程进行封装，使开发者只需要关注SQL本身，而不需要话费精力去 注册驱动，建立连接，创建statement，设置参数，获取结果集…

(3) MyBatis通过xml或注解的方式将要执行的各种statement配置起来，并通过java对象和statement中的sql进行映射，生成最终执行的sql语句，最后有MyBatis框架执行sql并讲结果映射成java对象并返回

(4) 种子，MyBatis对jdbc访问数据库的过程进行了封装，简化了jdbc代码，解决jdbc将结果集封装为java对象的麻烦

MyBatis组成

  1. mybatis-config.xml是MyBatis的核心配置文件，通过其中的配置可以生成SqlSessionFactory，也就是SqlSession工厂
  2. 基于SqlSessionFactory可以生成SqlSession对象
  3. SqlSession是一个既可以发送SQL去执行，并返回结果，类似于JDBC的Connection对象，也是MyBatis中至关重要的一个对象
  4. Executor是SqlSession底层的对象，用于执行SQL对象
  5. MapperStatement对象也是SqlSession底层的对象，用于接受映射（SQL语句中的参数），以及做输出映射（SQL查询的结果映射成相应的结果）

    使用MyBatis的原因

    (1) 使用传统的jdbc访问数据库

    [1] 使用JDBC访问，有大量的重复代码
    [2] JDBC自身没有连接池，会频繁创建连接和关闭连接，影响运行的效率
    [3] SQL是写死在程序中，一旦修改SQL，需要对类重新编译
    [4] 对查询SQL执行后返回的ResultSet对象，需要手动处理，有时候会特别麻烦

    (2）使用MyBatis框架访问数据库

    [1] MyBatis对JDBC进行了封装，可以简化JDBC代码
    [2] MYBatis自身支持连接池（也可以配置其他的连接池），因此可以提高程序的效率
    [3] MyBatis是将SQL配置在mapper文件中，修改SQL只是修改配置文件，类不需要重新编译
    [4] 对查询SQL执行后返回的ResultSet对象，MyBatis会帮我们处理，转换成java对象

总之，JDBC所有的问题（代码繁琐，有太多重复代码，释放资源，对结果的处理太麻烦等），在MyBatis框架中几乎都得到了解决

(4) mappers标签：用于导入mapper文件的位置，其中可以配置多个mapper，即可以多个导入mapper文件

7、MyBatis的占位符
(1) #{}占位符
① 相当于JDBC中的 ？ 是为SQL语句中的参数值进行占位，大部分情况下，都是#{},并且当#{}占位符是字符串或者日期类型的值进行占位，在参数值传过来替换占位符是，会进行转义处理

  1. 在mapper文件中：select * from emp where name=#{name}
  2. 在程序执行过程公，select * from emp where name =?
  3. 参数：替换后会自动添加单引号
    SQL语句中包含的参数值是传递过来的，在SQL语句中我们会通过#{}占位符进行占位，在SQL语句真正执行时，再将传递过来的值替换SQL语句中的占位符
    其实#{}就是JDBC中的？,因此为了安全考虑，在执行时，会对传递过来的字符串和日期类型的值进行转义处理
    (2) ${}占位符
    ① 是SQL片段（字符串）进行占位，将传过来的SQL片段直接拼接在${}占位符所在的位置，不会进行任何转义，（由于是直接讲参数拼接在SQL语句中，因此可能会引发SQL注入攻击问题）
    ② 需要注意：使用${}占位符为SQL语句片段占位时，即使一个占位符，也需要传一个参数，需要将参数先封装再传递
    ③ 如果在传递的时候不是要给参数值，而是一个SQL片段会怎么样？
  4. 例如在查询的时候，想动态传递查询的列
    a. Select #{} from emp;
    b. 此时传递过来的应该是一个SQL片段，不同于参数值，如果此时还用#{},也会像上面一样被转义处理 select ‘id,name,job’ from emp。这就不是我们希望看到的
    c. 如果不想让传过来的SQL片段被转义处理，而是直接拼接在SQL语句中，那么这里可以使用${}，比如：select ${    } from emp, 拼接之后就是 select id,name,job from emp
  5. 注意：在传递${}对应的值时，即使只有一个参数，也需要将值存储map中
  6. 大多数情况下还是使用#{}占位符，而${}多用途SQL片段进行占位
    8、Mapper接口开发
    (1) 以上是通过SqlSession对象调用方法进行增删改查操作，方法中需要闯入的第一个参数是一个字符串，该值对应的内容就是 namespace+id，通过这种方式，找到Mapper文件映射的SQL语句并执行
    (2) 这种方式由于传入的是字符串，很容易发生字符串拼写错误，且编译时，不会提示
    (3) 所以才有这种跟简单的方式，也是企业中最常用的方式，使用mapper接口开发
    (4) 注意：
    ① 创建一个接口，接口的全限定类名和mapper文件的namespace值要相同
    ② Mapper文件每条要执行的sql语句，在接口中要添加一个对应的方法，并且接口中的方法名和SQL标签的id值要相同
    ③ Mapper接口方法接受的参数类型，和mapper.xml中定义的sql的接受的参数类型要相同
    ④ 接口方法的返回值类型和SQL标签上的resultType即返回值类型相同（如果方法返回值是集合，resultType只需要制定集合中的泛型）
