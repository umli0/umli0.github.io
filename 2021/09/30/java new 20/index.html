做项目的开发门店管理

查询所有门店

(1)点击【门店管理】，访问DoorController，查询所有门店信息的方法，doorlist
(2) 调用service层，在service中再调用dao层查询所有门店信息，返回所有门店信息的List集合
(3)在doorList中，将返回的所有门店信息集合存入Model中，并转向door_list.jsp,将所有门店信息带到jsp页面进行显示

准备工作

（1）创建Door实体类
（2）创建Door实体类的映射文件DooeMapper
（3）dao层代码实现
（4）Service层代码实现

创建通用的页面跳转方法

@PathVariable 映射URL绑定的占位符
URL中的{XXX}占位符可以通过@PathVariable(“xxx”)绑定到操作方法的入参中，
如果，现在要访问/WEB-INF/pages/index.jsp，按照之前的简介，我们会在Controller中添加一个方法
如果有很多这样的jsp页面，需要我们访问怎么办？每一个jsp都要在Controller中添加一个方法，这么做就太繁琐了
@RequestMapping（“/index”）
public String xxx(){
return “index”;
}
我们可以发现 参数/后的内容，跟return的内容相同，我们可以提供一个通用的方法

删除门店信息

  • (1)在门店信息列表页面中，点击门店信息后的【删除】按钮，访问DoorController中的【doorDelete】方法，访问的同时讲所有要删除的门店id一同交给服务器
  • (2)在【doorDelete】方法中，通过参数接受门店id，调用service层，service层再调用dao层通过id删除指定的门店信息
  • (3)删除成功后，重定向到门店列表页面，显示所有门店（即重定向到DoorController的【doorList】方法），先查询所有门店，再将所有门店信息带到JSP显示
  • (4)需要注意的是，因为门店表和订单表存在关联关系，并且在订单表中添加了外键，因此如果该门店下有订单，门店不能删除（可以设计级联删除，在删除门店的同时，将订单表中的对应订单也删除）

新增门店信息

  • (1)在门店信息列表页面中，点击【新增门店】，跳转到门店新增页面【door_add.jsp】

  • (2)用户在【door_add.jsp】中输入新增的门店详细，点击【提交】将慢点信息提交给DoorController的【doorAdd】方法

  • (3)在【doorAdd】方法中，通过参数接受浏览器提交过来的门店信息，调用service层，service层再调用dao层将门店信息保存到数据库中

  • (4)新增成功后，重定向到门店列表页面，显示所有门店（即重定向到DoorController的【doorList】方法，先查询所有门店，再将所有门店信息带到JSP显示）

更新门店信息

  • (1)在门店信息列表中，点击门店信息后的【修改】按钮，访问DoorController中的【doorinfo】方法，访问的同时将所要查询的门店ID一同交给服务器
  • (2)在【doorinfo】方法中，通过参数接收门店id，调用service层，service层再调用到层查询制定门店信息（Door）并返回
  • (3)在【doorinfo】方法中，将返回的门店信息（door）存入model，并转向door_update.jsp,将门店信息带到jsp页面进行回显
  • (4)在原有门店信息基础上，对门店信息进行修改，点击【提交】将修改后的门店信息提交给DoorController的【doorUpdate】方法
  • (5)在【doorUpdate】方法中，通过参数接受浏览器提交过来的门店信息，调用service层，service层再调用dao层根据id修改门店信息
  • (6)修改成功后，重定向到门店列表页面，显示所有门店（即重定向到DoorController）的【doorList】方法，
