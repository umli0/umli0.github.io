状态管理

什么是状态管理

将浏览器与web服务器之间多次交互当作一个整体来看,并且多次交互说涉及到的数据的(即状态)保存下来

状态管理解决的两个问题

1.身份的识别
a.区分登录账号的个性设置
b.在企业中感觉员工的级别,设置不同级别的权限
2.状态的保护和维护

如何进行状态管理
客户端的状态管理方案

1.将状态保存在客户端(浏览器),cookie技术
Cookie一般存储的就是用户的个人信息(姓名,手机号,登录密码等)

服务器端的状态管理方案 Session

将状态保存在服务器端

Cookie技术

1.服务器临时保存在浏览器端的少量数据
2.Cookie的工作方式

  (1).当浏览器房访问服务器是,服务器会将少量数据以set-cookie消息头的方式发送给浏览器,浏览器会将这些数据保存下来
  (2).    当浏览器再次访问服务器时，会讲之前保存的数据以cookie消息头的方式发送给服务器，从而识别登陆用户

如何添加cookie

c.setMaxAge(40);//cookie的保存时间
resp.addCookie(c);//添加cookie

cookie的编码问题
Cookie只能保存合法的ascll字符,如果要保存非ascll字符,需要将中文转换成相应的ascll字符的表现形式

a) String URLEncoder.encode(String str,String charset);编码 添加cookie的时候
b)String URLDecoder.decode(String str,String charset);解码 获取cookie的时候

cookie的生存时间

如果添加cookie,没有设置生存时间,浏览器会将cookie保存在内存中,只要浏览器不管你,cookie就会一直存在,cookie就一直存在,如果浏览器关闭,cookie就删除

Cookie.setMaxAge(int seconds)
1)单位是秒
2)secondes的值可以是是以下三种情况
a.大于0:一般会报错cookie在硬盘上,超过制定时间,浏览器会删除
b.小于0:默认值
c.等于0:删除cookie

a.    不如，要删除cookie名为uid
a)    Cookie c = new Cookie(“uid”,””);
b)    c.setMaxAge(0)
c)    Resp.addCookie(c)

cookie的路径问题

1.浏览器向服务器上的某个地址发请求时，会比较要访问的地址的时候与cookie的地址匹配，只有匹配才会发送

2.服务器不是获取浏览器全部的Cookie，而是当前访问路径匹配下的所有cookie
cookie的默认路径就是该cookie当前的路径

cookie是浏览器发送请求后，服务器创建的，服务器创建cookie后发送至浏览器保存，后期有需要浏览器的cookie时，浏览器又把保存的cookie发送给服务器

Cookie的缺点

1.不安全，如果有敏感数据，必须以cookie的方式来保存，必须加密（加密算法，比如MD5）

2.可以被用户禁止

3.Cookie只能保存少量数据（4k）

4.浏览器能够保存cookie的数量也是有限制的（300个左右）

Session(会话)

session是什么呢

session是服务器为客户端创建的一个用来维护状态的对象

session的工作原理是什么

1）浏览器访问服务器时，服务器创建一个session对象（该对象有唯一的一个id,该对象称为session）
2）服务器将session以cookie的方式发送给浏览器
3）当浏览器再次访问服务器时，会将sessionid发送给服务器，服务器可以一句sessionid找到之前的session对象

浏览器访问服务器的servlet，服务器的servlet创建一个session对象，然后服务器发送session的ID给浏览器，浏览器就会保存Session的ID，然后浏览器又会发送session的ID给服务器，服务器就会根据Session的ID来寻找session

如何获得session对象

方一   HttpSession s=request.getSession(boolean flag)
1)当flag=true时，查看请求当中是否有sessionId，如果没有，就创建SessionId对象，如果有，则返回
2）当flag=false时，查看请求当中是否有sessionId,如果没有，返回null,如果有，那么就一句sessionid查找对应的session对象，如果找到了，则返回，如果没找到，返回null

方二   HttpSession s=request.getSession() -----request.getSession(true)

绑定数据相关的几个方法

1.绑定数据到session对象，setAttribute(String name,Object value)
2.依据绑定名找到绑定值，开始可能返回null
3.解除绑定removeAttribute(String name)

什么是session超时

1.服务器将空闲时间过长的session对象删除掉，一般服务器默认的超时限制是30分钟
