重定向

  • 重定向是指当浏览器请求一个URL时，服务器返回一个重定向指令，告诉浏览器地址已经变了，麻烦使用新的URL再重新发送新请求。
    通过响应，转换一个url地址，跳到一个新的页面
Response 会删除所有缓存数据 一般出现在，没有绑定数据的前提下，跳转页面
为什么不直接用直接跳转？
1、安全，保证跳转的页面的源文件 html，jsp不能被直接访问
2、为了统一管理，跳转方面的过程命名要规范化

转发的介绍

Forward是指内部转发。当一个Servlet处理请求的时候，它可以决定自己不继续处理，而是转发给另一个Servlet处理。

例如，我们已经编写了一个能处理/hello的HelloServlet，继续编写一个能处理/morning的ForwardServlet：

@WebServlet(urlPatterns = "/morning")
public class ForwardServlet extends HttpServlet {

    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        req.getRequestDispatcher("/hello").forward(req, resp);
//ForwardServlet在收到请求后，它并不自己发送响应，而是把请求和响应都转发给路径为/hello的Servlet，
//而后续请求的处理实际上是由HelloServlet完成的，这种处理方式称为转发（Forward）
    }
}

转发和重定向的区别


（1）转发涉及的各个web组件就可以共享一个request对象，重定向不可以
（2）重定向之后，浏览器地址栏的地址发生改变，转发不会
（3）重定向的地址是任意的，转发地址必须是同一个应用
（4）转发是一件事没完成，重定向是一件事完成了
如果要绑定数据时，使用转发

<%@include %>指令

告诉容器（tomcat），在将jsp文件转换成servlet的时候，将file属性指定的文件的内容插入到指令所在的位置
<%@include file=”***.jsp”%>

在某位置存在<%@include%>指令，就将指令中file属性指定的文件内容插入到该位置
