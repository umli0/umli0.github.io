订单操作

查询所有订单

开发步骤

  1. 点击订单管理，访问OrderController，查询所有订单信息 【orderList】

  2. 在【orderList】方法中调用service层，service层再调用dao层查询所有订单信息，返回所有订单信息的List集合

  3. 在【orderList】方法中，将返回的所有订单信息集合（List）存入model中，并转向order_list.jsp,将所有订单信息带到jsp页面进行显示

准备工作

  1. 创建实体类，用于封装所有的订单信息

  2. 在resources/mybatis/mapper目录下创建Order实体类的映射文件

  3. dao层代码实现
    (1)更新resources/mybatis/mapper/OrderMapper.xml，添加“查询所有订单信息”SQL
    (2)创建OrderMapper接口，添加查询所有订单方法

  4. service层代码实现

  5. controller层代码实现

  6. 在页面上展示所有订单列表

删除订单信息

(1)开发步骤

  1. 在订单信息列表中，点击订单信息后的【删除】按钮，访问OrderController中的【orderDelete】方法，访问的同时，将所要删除的订单id一同交给服务器
  2. 在【orderDelete】中，通过参数接收订单id，调用service层，service层再调用dao层通过id删除制定的订单信息
  3. 删除成功后，重定向到订单列表页面，显示所有订单

(2)dao层代码实现

  1. 更新OrderMapper.xml文件，添加“删除订单信息”
  2. OrderMapper接口添加deleteById方法，根据ID删除订单信息

新增订单信息

(1)开发步骤

  1. 在订单信息列表页面中，点击【新增订单】，查询所有门店，将所有门店信息带到订单新增页面【order_add.jsp】

  2. 在【order_add.jsp】中输入新增的订单信息，点击【提交】将订单信息提交给OrderController的orderAdd方法

  3. 在【orderAdd】方法中，通过参数接受浏览器提交过来的订单信息，调用service层，service层再调用dao层将订单信息保存到数据库中

  4. 新增成功后，重定向到订单列表页面，显示所有订单（重定向）
    (2)dao层实现

(3)更新OrderMapper接口，添加新增订单方法

(4)更新service层

(5)更新Controller层
修改”新增订单”链接地址，点击链接访问地址，查询所有门店信息，将所有门店信息转发到order_add.jsp中，并在下拉框中回显

更新订单信息

(1)开发步骤

  1. 在订单信息列表页面中，点击订单信息后的【修改】，访问OrderController的【orderInfo】方法，访问的同时，将所要查询的订单id一同提交给服务器

  2. 在【orderInfo】中，通过参数接收订单id，调用service层，service层再调用dao层通过id查询指定的订单信息【Order】并返回

  3. 在【orderInfo】方法中，将返回的订单信息（Order）存入Model中，并转向order_update.jsp,将订单信息带到jsp页面进行回显

  4. 在原有订单信息的基础上，对订单信息进行修改，点击【提交】将修改后的订单信息提交给OrderController的【orderUpdate】方法

  5. 在【orderUpdate】方法中，通过参数接收浏览器提交过来的订单信息，调用service层，service再调用dao层根据ID修改订单信息

  6. 修改成功后，重定向到订单列表页面，显示所有订单
